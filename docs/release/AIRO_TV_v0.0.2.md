# Airo TV v0.0.2

Airo TV v0.0.2 is a release-management and trust-readiness update for the Android TV variant of Airo.

## Highlights

- Published clean release asset names for APK and Play Store AAB downloads.
- Added a structured release-note format for Airo TV releases.
- Added SHA256 checksum publishing for release artifacts.
- Documented privacy, security, threat model, roadmap, and release asset expectations.
- Updated public download links and Airo TV release documentation.

## New Features

### IPTV

- Supports user-supplied M3U playlist URLs.
- Keeps IPTV content responsibility with the user; Airo TV does not bundle channels or playlists.

### Search

- Supports channel search by channel name.
- Includes validation coverage for the Music India search/play flow.

### Android TV

- Supports the Android TV package `io.airo.app.tv`.
- Supports both standard launcher and Leanback launcher entry points.
- Maintains Pixel/mobile fallback layout coverage for portrait and landscape.

### Chromecast

- Supports Cast discovery state messaging.
- Supports play, pause, stop, reload, new session, and volume controls.
- Documents the local-network requirements for Cast discovery and port reachability.

### Playback

- Supports IPTV channel playback from authorized playlist URLs.
- Keeps playback behavior scoped to user-provided streams.

### User Interface

- Documents required screenshots for Android TV Home, Player, Search, Playlist Import, Settings, Chromecast, and error states.
- Maintains accessibility label and tooltip verification for key controls.

## Improvements

- Release artifacts now use user-facing names:
  - `Airo-TV-v0.0.2.apk`
  - `Airo-TV-v0.0.2-Play-Store.aab`
- Release assets include `SHA256SUMS` for download verification.
- Release notes follow a stable, professional template suitable for future versions.
- README and public site copy point users to the tagged Airo TV release.
- Release documentation now includes privacy, security, roadmap, architecture, feature matrix, and reproducible-build references.

## Bug Fixes

- Fixed broken and misleading Airo TV download references that pointed to generic or missing release assets.
- Removed debug-looking public APK names from the Airo TV release flow.

## Performance

- The Android TV build continues to use the lightweight TV pubspec to avoid bundling unrelated heavy dependencies.
- Release builds continue to use icon tree-shaking and obfuscation with split debug information.

## Developer Changes

- Updated the Airo TV release workflow defaults for `v0.0.2`.
- Added SHA256 checksum generation to the release pipeline.
- Added a reusable Airo TV release template.
- Added trust documentation used by the README and release process.

## Breaking Changes

No breaking changes.

## Known Limitations

- No EPG support.
- No recording.
- No favorites sync.
- No cloud playlists.
- No multiple playlist merge.
- No bundled IPTV channels or media content.
- Cast discovery depends on `_googlecast._tcp` advertising and port `8009` reachability on the local network.
- Screenshot and demo-video assets are documented as required release assets, but must be captured from final release devices before store submission.

## Installation

### APK

Download `Airo-TV-v0.0.2.apk` from the GitHub release assets and install it on a compatible Android device or Android TV device that allows APK installation.

### Play Store

Upload `Airo-TV-v0.0.2-Play-Store.aab` to Play Console and promote it through the selected testing or production track.

## Legal Notice

This application is an IPTV player.

It does not provide television channels, media content, or playlists.

Users are responsible for supplying legally authorized IPTV playlists.
