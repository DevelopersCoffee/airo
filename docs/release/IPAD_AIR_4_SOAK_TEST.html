Qualification Analysis

The Airo TV app successfully qualified the memory budget on iPad Air 4. The Peak RSS (physical RAM footprint) during the entire playback soak test was 212.98 MiB, remaining well under the allocated budget threshold of 250 MiB.

The Dart heap footprint remained extremely stable. The drift over the 34 minutes of playback was minimal, indicating that the garbage collector is active and that no progressive Dart memory leaks exist. The integrated LeakTracker returned a leak count of 0 leaks, verifying that all widgets and controllers were successfully collected during channel switching.

Channel Switching Bug Fix

During active verification, a race condition was discovered when switching channels rapidly. The app threw a fatal platform exception: Bad state: No active player with ID 1.

The bug occurred because VideoPlayerStreamingService._disposeController() was an asynchronous operation. During the await suspension of the player disposal, the controller reference was still non-null. When the UI rebuilt (e.g. on channel change state updates), it attempted to build a VideoPlayer widget using this half-disposed controller.

We resolved this by setting the controller reference to null synchronously at the start of disposal, forcing the UI to immediately fall back to loading/placeholder layouts: 

Future<void> _disposeController() async {
  _liveEdgeDetector.detach();
  final oldController = _controller;
  _controller = null; // Set to null synchronously first
  if (oldController != null) {
    oldController.removeListener(_onControllerUpdate);
    await oldController.dispose();
  }
}
