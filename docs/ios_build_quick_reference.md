# iOS Build Quick Reference

## Current state

Airo is not SPM-only today.

The supported iOS dependency model is hybrid:

- Flutter engine/dependency generation may use Swift Package Manager where Flutter supports it.
- Flutter plugins that ship iOS podspecs still require CocoaPods.
- Do not remove the Podfile, skip `pod install`, comment plugins out, or swap in a reduced iOS-only pubspec to make a local build pass.

## Prerequisites

- macOS with Xcode installed.
- Flutter SDK matching this repository.
- CocoaPods available as `pod`.

Check locally:

```bash
xcodebuild -version
flutter --version
pod --version
```

If CocoaPods is missing, install it before building iOS. The exact install method depends on the machine setup, for example Homebrew or RubyGems.

## First-time setup

From the repository root:

```bash
make setup-ios
```

That target checks for Xcode and CocoaPods, runs `flutter pub get`, then runs `pod install` in `app/ios`.

Manual equivalent:

```bash
cd app
flutter pub get
cd ios
pod install
```

## Daily commands

From the repository root:

```bash
# Build release iOS app. Requires macOS, Xcode, and CocoaPods.
make build-ios

# Run on an iOS simulator/device selected by Flutter.
cd app
flutter run -d ios
```

Common manual build variants:

```bash
cd app

# Debug simulator build.
flutter build ios --simulator --debug --no-codesign

# Release device build. Requires valid signing configuration.
flutter build ios --release

# Profile build.
flutter build ios --profile
```

## Clean rebuild

Use this when switching branches or after dependency changes:

```bash
cd app
flutter clean
flutter pub get
cd ios
pod install
cd ..
flutter build ios --simulator --debug --no-codesign
```

If generated CocoaPods state appears stale, remove generated artifacts and reinstall:

```bash
cd app
rm -rf ios/Pods ios/Podfile.lock ios/.symlinks
flutter pub get
cd ios
pod install
```

## What not to do

- Do not force SPM-only for this app today.
- Do not treat a "CocoaPods not installed" message as harmless; CocoaPods is required for the current plugin set.
- Do not delete or ignore `app/ios/Podfile`.
- Do not comment plugins out to avoid CocoaPods.
- Do not replace `app/pubspec.yaml` with a reduced iOS-only manifest for normal builds.
- Do not use `app/pubspec_ios_spm.yaml` or `app/pubspec.yaml.backup`; those were local SPM-only experiment artifacts and are not part of the supported build path.

## File locations

```text
app/ios/
├── Podfile                    # CocoaPods integration for Flutter plugins
├── Podfile.lock               # Generated by pod install when present locally
├── Pods/                      # Generated CocoaPods checkout/build state
├── Runner.xcodeproj/          # Xcode project
├── Runner.xcworkspace/        # Xcode workspace used by Flutter/Xcode
├── Runner/                    # App source code
└── Flutter/                   # Flutter-generated iOS build configuration
```

Generated Flutter SPM files may appear under `app/ios/Flutter/ephemeral/` depending on Flutter configuration. They do not replace CocoaPods for this app's current plugin set.

## Troubleshooting

### CocoaPods is missing

Install CocoaPods, then rerun:

```bash
make setup-ios
```

### Pods are stale or missing

```bash
cd app
flutter pub get
cd ios
pod install
```

### Build fails after branch switching

```bash
cd app
flutter clean
rm -rf ios/Pods ios/Podfile.lock ios/.symlinks ios/Flutter/ephemeral
flutter pub get
cd ios
pod install
```

### Plugin module not found

First verify the plugin still exists in `app/pubspec.yaml`, then regenerate Flutter and CocoaPods state:

```bash
cd app
flutter pub get
cd ios
pod install
```

If the failure persists, inspect the plugin's iOS installation instructions and podspec support. Do not remove the plugin just to make the build green.
