# CV-033 Dogfood Prep (#889) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare everything needed for a real phone → real receiver hardware dogfood pass of CV-033 phone-hosted streaming: confirm the debug entry point still works, add a Wi-Fi teardown listener so the server doesn't rely solely on the 2-minute idle timeout, and scaffold the evidence document — without touching the streaming algorithm itself.

**Architecture:** `PhoneMediaCastHandoff` (`packages/platform_player/lib/src/services/phone_media_cast_handoff.dart`) owns a `PhoneMediaFileServer` instance and already has a `dispose()` method (line 189) that calls `server.stopServer()` (line 237). This plan adds a `connectivity_plus`-backed listener that calls the same teardown path proactively when the phone's Wi-Fi drops, instead of waiting for the existing idle-timeout mechanism to eventually expire the session.

**Tech Stack:** Flutter/Dart, `connectivity_plus` (already a dependency across `app/pubspec*.yaml` and `core_data`), `flutter_test`.

## Global Constraints

- Do not modify LAN bind allow-list, HTTP Range/suffix-range handling, constant-time token compare, or the idle/expiry lifecycle itself — CV-033's USP, already covered by `phone_media_large_file_soak_test.dart` and `phone_media_file_server_test.dart`. This plan only adds a new proactive-teardown trigger alongside the existing idle timeout, it does not replace or alter it.
- Diagnostics events (`onSessionEvent`) must never carry URLs, tokens, or file paths — same rule as the rest of `PhoneMediaFileServer`, applies to any new event this plan adds.
- Worktree: `.claude/worktrees/cv033-dogfood-889`, branch `worktree-cv033-dogfood-889`.

// Constructor gains:
PhoneMediaCastHandoff({
  // ...existing params...
  Stream<List<ConnectivityResult>>? debugConnectivityStream,
}) : _connectivityStream =
         debugConnectivityStream ?? Connectivity().onConnectivityChanged;

// Wherever the server is started (near line 153's PhoneMediaFileServer(...) construction):
_connectivitySubscription = _connectivityStream.listen((results) {
  if (results.every((r) => r == ConnectivityResult.none)) {
    onSessionEvent?.call('wifi_disconnected_teardown', const {});
    unawaited(server.stopServer());
  }
});

// In dispose() (line 189), before the existing stopServer() call:
await _connectivitySubscription?.cancel();
_connectivitySubscription = null;
