# v0.0.5 Milestone — Two-Layer Architecture: Media Engine ↔ Edge Intelligence SDK

Date: 2026-07-19
Status: Approved for milestone planning
Milestone: `v0.0.5` (GitHub milestone #8)

## Thesis

Matching the current IPTV feature ceiling makes Airo just another IPTV app. The
differentiator is a hard split into two independently evolving layers:

- **Airo TV (Media Engine)** — deterministic: rendering, playback, IPTV, EPG,
  DVR, MultiView, downloads, profiles, performance. Never knows about AI.
- **Edge Intelligence SDK** — intent, recommendations, automation,
  personalization, reasoning (on-device SLM). Never touches playback internals.

The only bridge is a typed command contract: the SDK emits structured intents;
the engine executes them.

```
Flutter UI
    │
Airo UI Framework
    │
Edge Intelligence SDK        (Intent, Recommendation, Automation, Search,
    │                         Profile, Notification engines; Knowledge Packs)
    │  IntentCommand contract (Intent / Entity / Filters / Sort / Confidence)
    ▼
Media Engine (Rust)          (Playback, IPTV, EPG, DVR, MultiView, Downloads,
    │                         Subtitles, Metadata, Provider plugins, Storage)
    ▼
Providers                    (IPTV, Jellyfin, Plex, Emby, YouTube, Local,
                              SMB/NFS, OTT connectors)
```

## Scope philosophy for 0.0.5

0.0.5 is the *first vertical slice* of this architecture, not the whole vision.
Each layer ships one convincing capability end-to-end:

1. The boundary exists in code (packages + contract), not just in docs.
2. Media Engine proves the performance story: 100k+ channels open < 1 s.
3. Edge SDK proves the intelligence story: one natural-language query resolves
   to a deterministic engine command on-device.

Everything else (sports intelligence, AI Director, scene intelligence, DVR
encryption, 16-way MultiView, marketplace) is deferred to 0.0.6+ and tracked as
Pro-line design work only.

## New work items for 0.0.5

### Layer 0 — The Boundary

**A. Edge Intelligence SDK scaffold + IntentCommand contract.**
New package `core_edge_intelligence` (or split under `packages/`): defines the
typed contract every intelligence feature must emit —
`{intent, entities[], filters[], sort, confidence}` — plus an
`IntentExecutor` interface implemented by the media-engine side. Static
analysis/CI rule: no import from intelligence packages into
`platform_player`/`platform_streams`/`core_media_routing`, and vice versa.

### Layer 1 — Media Engine

**B. Playlist engine v2 (Rust).** Memory-mapped incremental M3U parser, binary
playlist cache, SQLite channel index, lazy/background loading. Target: cold
open of a 100k-channel playlist to first rendered list < 1 s on Fire TV Stick
class hardware. Builds on CV-010 (#814) virtualized lists.

**C. Deterministic search engine (Rust).** Multi-field index: title, aliases,
genre, language, country, provider, tags. Prefix + fuzzy + rank. Pure engine —
no AI. Exposed over FRB; consumed both by UI search and by the SDK's intent
executor. Builds on CV-006 (#810).

**D. EPG multi-source merge.** Multiple XMLTV + provider EPG sources per
channel, duplicate removal, partial/live refresh, missing-program gap
handling. Extends `platform_epg`.

**E. MultiView v1.** 2- and 4-tile layouts, GPU-composited, focus-follows
audio, per-tile independent audio/subtitle tracks, tile swap. Supersedes
deferred CV-020 (#829) with a scoped v1.

### Layer 2 — Edge Intelligence SDK

**F. Intent Engine v1.** On-device SLM (existing `core_ai` LiteRT / Gemini
Nano infra) converts natural queries — "show every India match today",
"kids cartoons", "continue yesterday's movie" — into IntentCommand JSON;
engine executes via the search index + EPG. Confidence threshold falls back to
plain search. Zero AI in the playback path.

**G. Watch routines + Smart Collections v1.** Local pattern learning
(rule-based first, per #881): detect repeated time/channel/genre habits,
surface one-click routines and auto-generated collections ("Weekend Cricket",
"Kids Morning"). All on-device.

**H. Conversation memory v1.** "Continue where I stopped" / "show similar"
resolved from `core_watch_progress` + history; state synced cross-device via
existing sync engine.

**I. Intelligent notifications v1.** Context-aware alerts from EPG + user
patterns ("your team's match starts", "program you follow is live now").
Deterministic triggers in engine; phrasing/selection in SDK.

### Layer 3 — Airo Pro (design-only in 0.0.5)

**J. Pro AI line design brief.** AI Director, AI Highlights, Sports Command
Center, Media Concierge, Personal TV Channel — design docs + entitlement
gating plan only. No implementation this milestone. Ties into #885
classification pass.

## Non-goals for 0.0.5

- DVR/recording (CV-027 #831 stays in its own track)
- mpv backend (CV-030 #837 own track)
- 6/8/9/12/16-way MultiView, drag-resize layouts
- Scene intelligence (skip intro/ads), sports event switching
- Voice control, cross-device automation, car/watch targets
- Marketplace, signed packs, plugin SDK

## Success criteria

1. `core_edge_intelligence` package exists; CI import-boundary gate green.
2. 100k-channel playlist cold-open benchmark < 1 s recorded in
   `platform_benchmarks`.
3. One natural-language query flows SLM → IntentCommand → engine → playback,
   fully offline.
4. MultiView 2/4-tile usable on Fire TV with remote only.
5. All milestone issues labeled CE (`non-pro`) or Pro (`pro`) per #885.
