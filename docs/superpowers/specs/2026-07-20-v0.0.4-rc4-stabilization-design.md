## Per-item technical design

### #590 — Cast channel-switch stability

- Add monotonic `_loadGeneration` counter in `flutter_chrome_cast_controller.dart`. Every `loadMedia` bumps it and tags the request; the receiver-status handler drops any status whose generation is stale (< current).
- `iptv_cast_providers.dart:109` — skip `controller.connect(device)` when the target device is already the active connected device; call `load` only.
- Update `iptv_cast_notifier_test.dart:75`: same-device switch expectation becomes `load:next` only (no second `connect:tv-1`).
- New controller test: stale generation-1 status arriving after generation-2 starts must not change session state (issue's AUTO-002).
- TDD: red (update + add tests) → green (implement guard) → refactor.
- Manual verification (UC-001/002, AUTO-003 — Pixel 9 → BRAVIA/Chromecast, `iptv-org` playlist) is a manual step, batched with #889's hardware pass.
- Review: platform-architect (native bridge/cast controller shape) + playback-architect.

### #863 — CI build-web fix

1. `cd packages/core_native && flutter_rust_bridge_codegen generate`; diff output against committed `frb_generated.dart` / `frb_generated.web.dart`.
2. If regen matches committed code (upstream version disagreement, not stale codegen): bump the `flutter_rust_bridge` pin to a version whose web tear-off signature is `RustLibWire Function(ExternalLibrary)`-compatible, regen, re-verify.
3. Verify: `flutter run -d chrome -t lib/main_tv.dart` builds; CI's actual `flutter build web` command passes locally.
4. No new tests (build-config fix) — the `build-web` CI job passing is the acceptance signal.
5. Scoped diff: codegen output + pin bump only.
6. Review: platform-architect (core_native / FFI shape) + chief-release-devops-officer (CI).

## Process

- **Milestone**: reuse airo milestone #7 "Airo TV v0.0.4" (due 2026-07-30) for #590, #863, #889 — no duplicate milestone created. airo-pro #17/#20 stay unmilestoned (no v0.0.4-equivalent milestone exists there; not inventing one to match).
- **Review**: every PR passes CLAUDE.md's Correctness→Clarity→Consistency→Duplication→Tests→Performance checklist plus the relevant Engineering Council agent(s) listed per item above. codex-delegate hook auto-runs on every Edit/Write per existing global config — no extra manual step.
- **Testing**: TDD (red→green→refactor) for all logic changes; no test padding for pure config/codegen fixes (#863) or a straight vendored-code sync (#20 PR1).
- **Hardware/manual steps**: batched into one pass you run — #590 (UC-001/002 device verification), IAP Task 8, #889 (12-item matrix). I prep code + evidence-doc scaffolding; you execute and report results back for me to file follow-ups from.
- **Release cut**: after all in-scope items merge (or are explicitly deferred/blocked), cut the v0.0.4-rc.4 pre-release draft — same flow as the rc.3 cut (tag, changelog, draft release, orchestrator build).
- **Final report**: per item — merged (PR#, commit) / deferred (reason, target rc) / blocked (blocker, what's needed). rc.2 leftovers not attempted get filed explicitly for rc.5, not silently dropped.
